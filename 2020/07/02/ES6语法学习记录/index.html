3.简单的搜索关键字

3.1 老方法indexOf

  • indexOf方法将从头到尾地检索字符串 stringObject，看它是否含有子串 searchvalue。开始检索的位置在字符串的 fromindex 处或字符串的开头（没有指定 fromindex 时）。如果找到一个 searchvalue，则返回 searchvalue 的第一次出现的位置。stringObject 中的字符位置是从 0 开始的。

  • 如果要检索的字符串值没有出现，则该方法返回 -1。

  • 如果searchvalue为空字符串‘’，则返回0，在本例中表现为，若搜索关键字为空，(this.list[i].name.indexOf(this.keywords) != -1)判断为真，每项都push进result数组，即返回整个list。

4.闭包

4.1 概念

函数和对其周围状态（lexical environment，词法环境）的引用捆绑在一起构成闭包（closure）。也就是说，闭包可以让你从内部函数访问外部函数作用域。在 JavaScript 中，每当函数被创建，就会在函数生成时生成闭包。

[MDN中关于闭包的解读][https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Closures]

4.2 应用场景

  • 1.在外部获取函数中的变量 2.根据传入值的不同，创建多个函数（函数工厂），用这种方生成的多个函数，共享相同的函数定义，但是保存了不同的词法环境。类似的，还能根据传入值的不同，生成多个点击事件。

  • 2.匿名闭包：在循环绑定事件对象时，将循环体用立即执行函数包裹，也能达到闭包的效果，此时每个回调函数的绑定都对应一个循环变量。

  • 3.模拟私有方法：返回多个公共函数对象，在外部可以通过点选函数名来使用这些函数对象，从而实现对私有变量的修改。
  • 4.为函数的回调创建新的词法环境，正常情况下，回调函数会在循环绑定事件结束后进行调用，这样就会导致所有的回调函数都共享了最后一次循环的词法还环境，用闭包可以避免这一点
    使用注意：1.闭包会造成额外的性能损耗，所以不是为了特定的目的，应避免使用闭包，比如需要开放内部数据时，用闭包虽然能达到效果，但更建议将返回数据的方法定义在原型上

5.原型链

每个实例对象（ object ）都有一个私有属性（称之为 proto ）指向它的构造函数的原型对象（prototype ）。该原型对象也有一个自己的原型对象( proto ) ，层层向上直到一个对象的原型对象为 null。根据定义，null 没有原型，并作为这个原型链中的最后一个环节。

几乎所有 JavaScript 中的对象都是位于原型链顶端的 Object 的实例

6.var let const的区别

声明方式 变量提升 暂时性死区 重复声明 初始值 作用域
var 允许 不存在 允许 不需要 除块级
let 不允许 存在 不允许 不需要 块级
const 不允许 存在 不允许 需要 块级
- 变量提升的含义：变量的声明可以在访问之后，比如console.log(i);var i=1,正常输出不报错，结果是undefine，（变量的赋值不存在提升，等效于 var i;console.log(i);i=1;）,但是换成其他两种声明方法就会直接报错。
- 暂时性死区： 如果在区块内用let或者const声明了某个变量。这个变量会被锁定在这个区块中（即使外层有这个变量的声明，但在块作用域中访问到的始终是内层声明的变量）
var tmp = 123;

if (true) {
tmp = ‘abc’;//报错，Uncaught ReferenceError: tmp is not defined
let tmp;
}
- 重复声明：指的是相同作用域内，let const声明过的变量不允许重复声明
- 初始值： const声明的是只读变量。所以声明时需要初始值
- 作用域： var声明的变量只有函数作用域和全局作用域，有时候会出现块内外的变量覆盖，典型的比如for循环，用于计数的变量i会泄露，还有if条件语句，根据不同情况对外层变量赋值，可能会发生覆盖。
var tmp = new Date();//处于全局作用域

function f() {
console.log(tmp);//处于函数作用域
if (false) {
var tmp = ‘hello world’;
}
}

f(); // undefined

7.null和undefine的区别

7.1 undefined

字面意思就是：未定义的值 。这个值的语义是，希望表示一个变量最原始的状态，而非人为操作的结果 。 这种原始状态会在以下 4 种场景中出现：
【1】声明了一个变量，但没有赋值
【2】访问对象上不存在的属性
【3】函数定义了形参，但没有传递实参
【4】使用 void 对表达式求值

7.2 null

null 的字面意思是：空值 。这个值的语义是，希望表示 一个对象被人为的重置为空对象，而非一个变量最原始的状态 。 在内存里的表示就是，栈中的变量没有指向堆中的内存对象。
当一个对象被赋值了null 以后，原来的对象在内存中就处于游离状态，GC 会择机回收该对象并释放内存。因此，如果需要释放某个对象，就将变量设置为 null，即表示该对象已经被清空，目前无效状态。

typeof null== ‘object’
null 有属于自己的类型 Null，而不属于Object类型，typeof 之所以会判定为 Object 类型，是因为JavaScript 数据类型在底层都是以二进制的形式表示的，二进制的前三位为 0 会被 typeof 判断为对象类型，而 null 的二进制位恰好都是 0 ，因此，null 被误判断为 Object 类型。

ECMAScript 规范认为，既然 null 和 undefined 的行为很相似，并且都表示 一个无效的值，那么它们所表示的内容也具有相似性，即有

undefined == null; // true
不要试图通过转换数据类型来解释这个结论，因为：

Number(null); // 0
Number(undefined); // NaN

// 在比较相等性之前，null 没有被转换为其他类型
null== 0 ; //false

8. js中new做了哪些事情？

  • 先理解想要创造一系列部分属性相同的对象，过程是：
    1.在构造函数中，先声明一个临时对象
    2.绑定这个临时对象的原型
    3.在构造函数中return临时对象
    4.在原型上定义公共属性

  • 用new关键字构建构造函数简化了以上操作
    1.在构造函数中直接使用this指向临时对象
    2.构造函数名.prototype直接指向每个构建出来的对象的原型，不需要绑定了
    3.不需要return了，new 构造函数直接返回构造出来的对象
    4.原型的名字也被规定,统一用prototype表示

  • 注意：构造函数中 this 关键字没有任何的值。
    this 的值在函数调用实例化对象(new object)时创建。即指向实例对象。

9.promise对象

9.1 定义

promise是通过new promise(function(resolve,reject){…})构造出来的一个对象，构造函数的两个参数分别是resolve和reject。它们是两个函数，由 JavaScript 引擎提供，不用自己部署。

9.2 构造函数的参数

resolve函数的作用是，将Promise对象的状态从“未完成”变为“成功”（即从 pending 变为 resolved），在异步操作成功时调用，并将异步操作的结果，作为参数传递出去；reject函数的作用是，将Promise对象的状态从“未完成”变为“失败”（即从 pending 变为 rejected），在异步操作失败时调用，并将异步操作报出的错误，作为参数传递出去。

9.3 promise实例对象的方法

promise实例生成以后，可以用then方法分别指定resolved状态和rejected状态的回调函数（value和err），当promise对象的状态从pending转为确定态之后，then方法绑定的回调函数就会被触发。
可让一个函数返回一个promise对象，使其拥有异步的功能。（用这种方法定义的函数调用后可以之间使用.then方法）

  • 注意：.reject是Promise构造函数的方法，返回一个带有拒绝原因的Promise对象

  • .catch是promise实例对象的方法，用于捕捉reject参数传回来的错误信息。

注意 :promise是异步函数，若要修改异步函数中所使用的变量的值，在异步函数外层修改，会导致异步函数中使用的是修改过的值。原因是修改变量值这个操作写在异步函数之外，是一个同步操作，会比异步先执行，导致异步函数中还没调用该值，修改已经完成了
