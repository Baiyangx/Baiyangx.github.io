# logistic regression (basic notation)

an algorithm for binary classification

eg. we have

  • an input of an image
  • output a label Y to recognize this image as being either a cat -> output one non cat -> output with zero

an image: suppose three separate matrices corresponding to the red green and blue color channels of this image assume that the input images is 64 pixels * 64 pixels, then we get 36464 pixel intensity values

to turn the pixel intensity values into a feature vector, we need to unroll all of these pixel values into a input feature vector X, finally we should get a long list feature vector listing out all the red, green and blue pixel intensity values of this image. Therefore, the total dimension of this vector x will be 64 by 64 by 3 (the total number we have in all of these matrices, 12288).

# notation (how we group the training examples input X into matrix )

use nx = 12288 to represent the dimension of the input features X (for brevity we just use the lowcase n to donate it, note that n = nx)

a single training example is represented by a pair (x, y)

x ∈ R^nx , means x is an nx dimensional feature vector, and every dimension contain a R(real number)

y ∈ {0,1}

our training set will comprise m training examples: ((x1,y1),(x2,y2),(x3,y3),...,(xm,ym)) m or M subscipt train or M subscript test : the number of training examples

a more compact notation: see the image

X.shape (nx,m) represent that X is a nx by m dimensional matrix

Y = [y1,y2,y3,...,y4]
Y ∈ R^(1xm) Y.shape = (1,m)

# logistic Regression (sigmoid function)

Y hat = p(y=1|x) the chance that y is equal to 1

the parameters of logiic regression will be: w ∈ R^nx, b ∈ R

for linear function: y hat = wTx+b w transpose X plus B

sigmoid function: Output y hat = sigmoid(wTx+b)

sigmoid function: 1/(1+e^-2)

when we try to learn logic regression, our job is to learn parameters W and B that Y hat becomes a good estimate of the chance of Y being equal to 1

the cost function: to measure that how well our algorithm is doing

# why we dont use squared function as cost function

? we can define the loss when our algorithm outputs Y hat and the truth label is y maybe the squared error or one half square, but people don't usually do this because when you come to learn the parameters you find that the optimizatin problem becomes non convex so you end up with ostentation problem it would not talk local optima so gradient descent may not find a global optimum

squared error seems like it might be a reasonable choice except that it makes gradient descent not work well so, we actually define a different loss function that plays a similar role as squared error that will give us an optimization problem that is convex
